配置settings.xml

使用的maven配置文件settings.xml文件位于conf/目录下

settings.xml内相关字段说明
字段名称 使用说明
localRepository 配置本地仓库地址
server 服务器相关的配置，比如：自建的私库服务器的登录信息。(可用于上传组件)
mirror 服务器的镜像。mirrorOf内设置需要镜像的仓库id，*表示所有仓库，url设置目标地址。
profile 每个profile 对应一套自定义配置。id配置的唯一Id。repositories组件仓库信息。pluginRepositories插件仓库信息。每个repository表示一个仓库。id表示该仓库id。与mirror配置中mirrorOf呼应。如果id设置为central则覆盖父pom文件内central仓库。(父pom在maven-model-builder-3.8.4.jar文件中) url表示仓库地址。releases与snapshots分别配置发行版与测试版的信息。enabled表示是否允许下载。updatePolicy表示更新策略。always表示一直更新(只对snapshots有效)、daily表示日更新(默认)、interval:X表示间隔分钟更新、never不更新。checksumPolicy可选项，针对下载的组件的校验策略：ignore, fail, warn。

项目pom的配置说明

pom.xml内相关字段说明
字段名称 使用说明
modelVersion 表示pom文件版本。
groupId 表示组织名称。
artifactId 表示项目名称。
version 表示版本。
packaging 打包形式。pom, jar, maven-plugin, ejb, war, ear, rar
dependency 项目中的所有依赖。groupId表示组织名称。artifactId表示项目名称。version表示版本。3者联合定义组件唯一坐标。type依赖的类型，默认是jar。classifier可以是任何字符串，使用时拼接在version后面定位及获取组件(例如：com.abc.helloworld-lib-1.0-jdk8.jar，这里设置的classifier是jdk8)。scope类或包的可见性。设置compile值时编译、测试、运行可见。设置provided值时编译、测试可见。设置runtime值时测试、运行时可见。设置test值时测试时可见。设置system值时编译、测试时可见。
distributionManagement 组件发布仓库配置管理。repository表示发行版仓库配置。id仓库唯一标识。name方便说明的仓库名称。url仓库的完整地址。snapshotRepository表示测试版仓库的配置。在maven的settings配置中可以设置登录信息(用于仓库读写)。
